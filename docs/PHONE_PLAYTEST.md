# CRYPTOBILL — плейтест на телефоне (со звуком)

Надёжный луп «поправил код → прокатался на телефоне со звуком». Это пункт #2 роадмапа
стабилизации (`docs/STABILIZATION_PLAN.md`). Без него мы чиним вслепую.

Телефон и Мак должны быть в **одной Wi-Fi сети**.

---

## 1. Какая команда запускает сервер

В терминале, из корня проекта:

```bash
./serve.sh
```

Скрипт сам находит LAN-IP Мака, биндит сервер на `0.0.0.0:4173` и печатает точный URL для
телефона (и QR-код, если установлен `qrencode`). Стоп — `Ctrl+C`.

Порт занят? Возьми другой:

```bash
PORT=4174 ./serve.sh
```

> Почему скрипт, а не «просто python»: IP Мака **меняется** (за время проекта был
> `192.168.1.128` → `.142` → сейчас `192.168.88.132`, и активный интерфейс уехал с `en0` на
> `en10`). Хардкодить URL бессмысленно — `serve.sh` каждый раз печатает актуальный.
> `python3 -m http.server 4173` без `--bind 0.0.0.0` слушает только localhost и **с телефона
> не виден** — именно поэтому LAN-линк раньше отваливался.

## 2. Какой URL открывать на телефоне

Формат:

```
http://<IP-Мака>:4173/
```

Точный адрес печатает `serve.sh` при старте — бери оттуда. Сейчас это, например:

```
http://192.168.88.132:4173/
```

`/` достаточно: корневой `index.html` — это симлинк на `toys/chartrider.html`.
Полный путь `http://<IP>:4173/toys/chartrider.html` тоже работает.

**Всегда жёсткий рефреш после правок** (иначе телефон покажет старую сборку из кэша):
потяни страницу вниз для перезагрузки, или закрой вкладку и открой заново.
Проверка свежести → см. §6 (лейбл `build`).

## 3. Что проверить, если телефон не подключается

По порядку:

1. **Одна сеть.** Телефон и Мак на одной Wi-Fi (не на мобильном интернете, не на гостевой/
   соседней сети). Это причина №1.
2. **Сервер жив.** В терминале должно висеть `serve.sh` (не закрыт, нет `Ctrl+C`).
3. **IP свежий.** Возьми URL из текущего вывода `serve.sh`, а не из старой заметки — IP меняется.
4. **Локально работает?** На Маке открой `http://localhost:4173/` — если тут пусто, проблема в
   сервере/папке, а не в сети.
5. **Файрвол Мака.** Системные настройки → Сеть → Брандмауэр: либо выключи на время теста, либо
   разреши входящие для `python3` (всплывёт запрос при первом запуске — нажми «Разрешить»).
6. **«Изоляция клиентов» на роутере** (AP/client isolation) — некоторые гостевые Wi-Fi не дают
   устройствам видеть друг друга. Переключись на основную сеть.
7. **Порт занят** (`Address already in use`) — запусти `PORT=4174 ./serve.sh` и открой URL с `:4174`.
8. **Проверка достижимости:** на телефоне в браузере набери `http://<IP>:4173/` руками; если
   «не отвечает» — это сеть/файрвол (пункты 1/5/6), если «404» — путь/папка.

## 4. Как проверить, что звук работает

Звук — WebAudio, он **стартует только после первого касания экрана** (так устроены мобильные
браузеры). Поэтому:

1. Открой игру, **тапни** (старт заезда) — должен пойти **рёв двигателя**, который меняет тон
   со скоростью. Это самый явный признак, что звук жив.
2. Тихо? Проверь:
   - **Громкость телефона** поднята.
   - **iPhone: боковой переключатель «без звука» (mute)** — на iOS он глушит WebAudio. Выключи
     беззвучный режим.
   - **В игре не выключен звук:** пауза (кнопка ‖ справа сверху) → строка «ЗВУК: ВКЛ». Тоггл
     звука — там же, либо клавиша `M` на десктопе.
3. Что должно звучать в CORE-режиме: рёв эндуро-двигателя (питч растёт со скоростью), sfx на
   прыжок/приземление/краш, плюс фоновая синтвейв-музыка. Громкости пока **не выверены на слух** —
   это как раз то, что ты оцениваешь живьём (раньше звук «проверялся» только на «нет ошибок»).

> Превью на Маке (preview-инструмент) **беззвучное и throttled** — звук и фил касаний оттуда
> проверить нельзя в принципе. Только реальный телефон. Ради этого вся эта дока.

## 5. Как провести 5-минутный плейтест

Цель одного захода — ответить на главный вопрос: **ядро (езда + погоня за дампом) само по себе
весело, тянет жать «ещё раз»?**

1. **0:00** — `./serve.sh`, открой URL на телефоне, жёсткий рефреш. На титуле убедись, что
   снизу стоит свежий `build …` (§6) и подпись «CORE TEST · только езда и погоня».
2. **0:00–0:30** — один заезд «на разогрев», просто почувствовать управление (газ/тормоз/прыжок,
   наклон в воздухе). Звук должен играть (§4).
3. **0:30–4:00** — **8–10 заездов подряд**, рестарт одним тапом после смерти. После каждой
   смерти 2 секунды на заметку. Конкретно следи за:
   - **Управление:** газ/тормоз/прыжок отзывчивы? наклон в воздухе понятен? нет ли залипаний/
     лишних тапов?
   - **Честность дампа:** ловит честно или «из ниоткуда»? можно отыграться после одной ошибки?
     (смотри поле **дамп Nm** в оверлее — видно, насколько он близко).
   - **Loop-out/краш:** понятно, за что умер? телеграф (красный значок вращения) успел предупредить?
   - **FPS** в оверлее — держится ~60 или проседает (где именно)?
   - **«Ещё один»:** ловишь ли себя на том, что жмёшь рестарт **не думая**, чтобы побить
     дистанцию? Это и есть ответ.
4. **4:00–5:00** — собери впечатления в **два РАЗДЕЛЬНЫХ списка**, как в плане:
   - **(а) Баги** — что сломано/глючит/тормозит (с цифрами из оверлея, если поймал).
   - **(б) Фил** — что не так по ощущениям (управление/дамп/скучно/непонятно/раздражает).
   - И прямой ответ: **тянуло на «ещё раз» или нет.**

Кидай оба списка в чат — дальше работаем точечно по ним, новых систем не добавляем.

## 6. Отладочный оверлей и лейбл сборки (встроены в игру)

**Оверлей** — компактная плашка сверху по центру, видна на телефоне всегда. Поля:

```
🐞 60fps · 120км/ч · 9.8vx        ← FPS · скорость(км/ч) · скорость(vx, для тюнинга физики)
   840m · дамп 18m                ← дистанция · зазор до дампа (зелёный далеко / жёлтый / красный близко)
   AIR · CORE ON · 2026-06-17·b2  ← состояние · режим · сборка
```

- **Состояние:** `GROUNDED` (на земле) / `AIR` (в воздухе) / `CRASHED` (разбился) /
  `TITLE` / `PAUSED`.
- **дамп Nm** меняет цвет по близости: зелёный (далеко) → жёлтый → **красный** (вот-вот поймает).
- **Свернуть/развернуть:** тапни по плашке (на телефоне) или клавиша `F` (десктоп). Выбор
  запоминается. Свёрнутая = просто `🐞` в углу.

**Лейбл сборки** `build 2026-06-17 · b2` — внизу титульного экрана и в оверлее. После каждой
правки кода он меняется. **Если после рефреша на телефоне лейбл старый — кэш не сбросился,
рефрешни ещё раз.** Это твоя гарантия, что играешь свежую сборку.

> Оверлей и лейбл — чистый инструментарий, на геймплей не влияют. FPS в Mac-превью показывает 0,
> потому что вкладка throttled; на реальном телефоне в фокусе — настоящие ~60.
