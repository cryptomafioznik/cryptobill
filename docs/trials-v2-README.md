# CRYPTOBILL trials25d — articulated asset pack v2

Исправленный комплект для следующего этапа. Старый пакет не удалён.

## Состав

- 6 корпусов с позами райдера, без колёс, вилки и маятника;
- отдельные круглые переднее и заднее колёса;
- отдельные вилка и задний маятник;
- отдельные crash bike / crash rider;
- runtime-версии и source 1024;
- два бесшовных материала трассы, торцы разрывов и предупреждающие шевроны;
- отдельное небо, солнце, по два варианта дальнего/среднего города и пальм;
- два состояния разрушения ящика, барьер целый/сломанный, шипы, старт, финиш и пройденный checkpoint;
- базовые pickups/объекты из v1 в `objects/base_from_v1`.

## Важное ограничение

Генеративная графика не гарантирует математически точные суставы. Координаты в `asset-pack.json` — стартовая калибровка. Claude обязан вывести debug-точки и один раз подстроить их в пределах примерно 4–12 px. После калибровки координаты фиксируются и больше не вычисляются по alpha bounds.

## Правильная сборка байка

Порядок слоёв:

1. заднее колесо;
2. задний маятник;
3. корпус с райдером;
4. передняя вилка;
5. переднее колесо;

Вся сборка вращается на `physicsAngle`. Колёса дополнительно вращаются вокруг собственных ступиц. Вилка и маятник получают ход подвески из физических wheel contacts.

Позы `body_rider_forward` и `body_rider_back` меняют только чтение переноса веса. Они не должны сами задавать угол рамы. Для wheelie используется физический угол всего rig.

Не переключать позы чаще чем раз в 80–120 мс. Использовать короткий crossfade или hysteresis.

## Трасса

Форма трассы всегда строится из физической `terr/front`-линии. PNG являются только материалами. На каждом segment quad применять отдельный affine transform; обычный screen-space `createPattern()` не следует наклону трассы.

На острых стыках использовать bevel join и miter limit. На разрывах рисовать отдельные `track_gap_cap_left/right`; соседние полигоны через пропасть не соединять.

## Окружение

Небо действительно бесшовно по X. Город и пальмы являются модулями A/B: чередовать их, менять промежутки и небольшое масштабирование. Не ставить один и тот же модуль подряд. Foreground-пальмы рисовать после байка, но не перекрывать важную зону дольше 250 мс.

## Что рисуется кодом

Контактная тень, мелкие искры, пыль, speed lines, glow, траектория debug, screen shake, HUD и состояния кнопок. Не превращать их в дополнительные большие PNG.

## Проверка перед интеграцией

- показать steering/rear axle joints поверх изображения;
- ступицы колёс совпадают с axle points;
- fork/swingarm не отрываются при полном ходе;
- смена body pose не сдвигает chassis pivot более чем на 3 px;
- колёса вращаются без биения;
- texture UV не меняет collider;
- торцы разрыва не соединяются между собой.
